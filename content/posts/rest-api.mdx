Una REST API es un conjunto de reglas que definen como se comunican dos aplicaciones. 
Estas reglas son las que permiten que una aplicación pueda enviar una petición a otra 
y que esta pueda responder de manera correcta, sin importar el lenguaje de programación
o el sistema operativo que se esté utilizando.

<Callout emoji="❔">
También se le conoce como RESTful API o RESTful web service y es un tipo de API.
</Callout>

## ¿Qué es una API?
El término API es un acrónimo de Application Programming Interface, que en español significa
Interfaz de Programación de Aplicaciones,  La mayoría de las aplicaciones para empresas 
deben comunicarse con otras aplicaciones internas o de terceros para llevar a cabo varias tareas.

**Por ejemplo**: una aplicación de comercio electrónico debe comunicarse con un proveedor de
pagos para procesar los pagos de los clientes. Una aplicación de gestión de proyectos
debe comunicarse con una aplicación de seguimiento de errores para registrar los errores
que se encuentran durante el desarrollo. Una aplicación de gestión de relaciones con los
clientes (CRM) debe comunicarse con una aplicación de correo electrónico para enviar
correos electrónicos a los clientes.

## Término REST: ¿Qué significa?
REST es un acrónimo de **Representational State Transfer**, que en español significa Transferencia
de Estado Representacional. REST es un estilo de arquitectura de software que se utiliza
para crear servicios web. Los servicios web RESTful son servicios web que siguen el estilo
de arquitectura REST.

REST no es un protocolo ni un estándar, sino más bien un conjunto de límites de arquitectura. 
Los desarrolladores de las API pueden implementarlo de distintas maneras.
Cuando el cliente envía una solicitud a través de una API de RESTful, 
esta transfiere una representación del estado del recurso requerido 
a quien lo haya solicitado o al extremo. La información se entrega por medio de HTTP 
en uno de estos formatos: JSON (JavaScript Object Notation), HTML, XLT, Python, PHP o texto sin formato. 
JSON es el lenguaje de programación más popular, ya que tanto las máquinas 
como las personas lo pueden comprender y no depende de ningún lenguaje, 
a pesar de que su nombre indique lo contrario.

## Conclusiones
- Una REST API es un conjunto de reglas que definen como se comunican dos aplicaciones.
- REST es un estilo de arquitectura de software que se utiliza para crear servicios web.
- Los servicios web RESTful son servicios web que siguen el estilo de arquitectura REST.
- REST no es un protocolo ni un estándar, sino más bien un conjunto de límites de arquitectura.

Recuerda estar al tanto de estos términos para crear tu propia REST API.
